# Agent instructions

## Bug find and fix

When asked to find and fix bugs (or debug production/server errors):

1. Investigate and report what is wrong: symptoms, likely root cause, affected files or routes, and how you would fix it.
2. Do not change code until the user explicitly asks to apply changes (for example: “go ahead”, “implement”, “make the fix”).
3. If they only want analysis, stop after the report.

Exception: if the user clearly instructs to find, fix, and commit (or similar) in the same message, you may implement after a short summary.
